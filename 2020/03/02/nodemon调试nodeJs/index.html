    nodemon调试nodeJs

    日积
    创建时间:2020-03-02 00:00
    阅读:
    1. 介绍
    2. 配置
    3. 使用

    介绍

    首先开发不能调试打断点那将会耗费很多的精力的，如果你是使用vscode开发，那么配置调试将是很简单的事

    配置

    首先在vscode的调试配置中添加配置

    {
    // 使用 IntelliSense 了解相关属性。 
    // 悬停以查看现有属性的描述。
    // 欲了解更多信息，请访问: https://go.microsoft.com/fwlink/?linkid=830387
    "version": "0.2.0",
    "configurations": [
        {
            "type": "node",
            "request": "launch",
            "name": "nodemon",
            "runtimeExecutable": "nodemon",
            "program": "${workspaceFolder}/nodeScript/server.js",//配置自己启动文件即可
            "restart": true,
            "console": "integratedTerminal",
            "internalConsoleOptions": "neverOpen",
            "skipFiles": [
                "<node_internals>/**"
            ]
        }
    ]
}

    使用

    在package.json中scripts配置

    "scripts": {
    "dev:client": "vue-cli-service serve",
    "dev:server": "cross-env NODE_ENV=development nodemon nodeScript/server.js",
    "dev": "concurrently \"npm run dev:client\" \"npm run dev:server\""
  }

    注意：使用concurrently整合的时候要讲命令进行转义
