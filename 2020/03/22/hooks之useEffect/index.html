    useEffect

    栗子

    import React, {usestate, useeffect} from react" 
function Example() {
    const [count, setCount] = usestate(0);

    useeffect(()=> {
        document title = 'You clicked scount times'; 
    }

    return (<div> 
        <p>You clicked count) times</p>
         <button onclick={()=>{setCount(count + 1)}> Click me </button> 
     </div>);
 }

    示例代码

    介绍

    useEffect来处理副作用，也具备生命周期函数

    结合三个class组件生命周期函数

    • componentDidMount

    • componentDidUpdate

    • componentWillUnmount

    render：首先介绍一下render这个最最关键的，唯一必须实现的lifecycle

    class组件生命周期图谱

    在对应的三个关键阶段中触发

    • 挂载：组件实例被创建并插入 DOM 中时

    render()
componentDidMount()

     • 更新：当组件的 props 或 state 发生变化时会触发更新

    render()
componentDidUpdate()

    • 卸载：当组件从 DOM 中移除时

    componentWillUnmount()

    用途

    通过使用这个 Hook，告诉 React 组件需要在渲染后执行某些操作

    常见用法

    *如何实现挂载回调？

    使用第二个参数，传入一个空数组(［］)，如：

    useEffect(() => {
      document.title = `You clicked ${count} times`;
}, []);

    分析：

    我觉得这点很有必要，因为经常我们需要在挂载后就执行一次，然后再render之后就不执行了

    在effect 返回一个函数，React 就会在卸载时调用它，如：

    分析

    简洁：

    不需要单独的为清除操作设立一个effect，直接和添加在同一个地方执行，因为添加和清除订阅的紧密性。

    卸载回调中往往要做什么？

    清除副作用

    什么副作用需要清除？

    例如订阅外部数据源，定时器。这种情况下，清除工作是非常重要的，可以防止引起内存泄露

    *如何实现刷新回调?

    单独实现：这是个比较罕见的使用场景。如果你需要的话，你可以 使用一个可变的 ref 手动存储一个布尔值来表示是首次渲染还是后续渲染，然后在你的 effect 中检查这个标识。（如果你发现自己经常在这么做，你可以为之创建一个自定义 Hook。）

    延迟加载

    effect 的执行时机与 componentDidMount、componentDidUpdate 不同的是，在浏览器完成布局与绘制之后，传给 useEffect 的函数会延迟调用。这使得它适用于许多常见的副作用场景，比如设置订阅和事件处理等情况，因此不应在函数中执行阻塞浏览器更新屏幕的操作。

    需要同步加载请使用useLayoutEffect

    小结

    紧密融合多个api虽好，但要区分

    1. 使用UseEffect第二个参数(依赖数组)，传入空数组来区分挂载回调

    2. 返回一个函数实现卸载回调

    3. 默认就包含了刷新回调，如若单独实现需借助useref(进阶)，但不常用

    4. 它是延迟加载的
